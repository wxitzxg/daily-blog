刚刚看见发了个新模型 KAT-Coder-Pro V2。

这么一说我就来劲了，抱着好奇心想去做几个简单测试，看一下它生成页面上的美学水平。

  

我一上来给的需求也很随意。

  

下面这个界面，我只丢了一句话给：“帮我制作一个电商 Landing Page 页面，主要是售卖瑜伽服套装，然后面向欧美客户，整体品牌需要高级一些。”

你会发现它给你自动配上了一些参考图并用层次的关系去梳理的更美。

  

同样的，我提出第二个需求的时候，也是很简单的一段话，“我最近要上线一个语法的AI硬件，主要是做的面向日本的市场，然后受众用户是学生 请你给我设计个 Landing page 页面，整体需要高级一些。”

最后生成出来的时候，这个页面效果我当时的感觉有点微妙，在版式统一、视觉节奏、以及图文关系上，是有控制力的。

  

在美学上，不是简单地把内容排进去，而是直接基于我的一句话描述，有一种「已经帮你想过」的感觉，我觉得这个 KAT-Coder-Pro V2 模型是真的站住了脚。

并且以上代码都是一轮对话直接生成。

  

做到这里，我其实已经觉得还不错了，我特意去看了下它的美学，这个模型是基于研发设计团队深厚的人文视觉与前端积淀，推出由专业设计师与合作团队校准完成的「KAT 美学 Benchmark」。

![图片](data:image/svg+xml,%3C%3Fxml version='1.0' encoding='UTF-8'%3F%3E%3Csvg width='1px' height='1px' viewBox='0 0 1 1' version='1.1' xmlns='http://www.w3.org/2000/svg' xmlns:xlink='http://www.w3.org/1999/xlink'%3E%3Ctitle%3E%3C/title%3E%3Cg stroke='none' stroke-width='1' fill='none' fill-rule='evenodd' fill-opacity='0'%3E%3Cg transform='translate(-249.000000, -126.000000)' fill='%23FFFFFF'%3E%3Crect x='249' y='126' width='1' height='1'%3E%3C/rect%3E%3C/g%3E%3C/g%3E%3C/svg%3E)

  

我在接入这个模型测试的时候，甚至在他们的后台看到了一个 KAT 前端生成与美学评估白皮书，看来这个是真的有把这个「美学模型」认真的以一件很专业的事情在做。

  

KAT 这次的做法，是直接跳出算法评测，引入设计师参与，构建了一套专门针对“无参考创作”的美学 Benchmark。

  

这套标准不再只是看单个元素，而是从配色、布局、元素设计、整体风格等多个维度去评估。

![图片](data:image/svg+xml,%3C%3Fxml version='1.0' encoding='UTF-8'%3F%3E%3Csvg width='1px' height='1px' viewBox='0 0 1 1' version='1.1' xmlns='http://www.w3.org/2000/svg' xmlns:xlink='http://www.w3.org/1999/xlink'%3E%3Ctitle%3E%3C/title%3E%3Cg stroke='none' stroke-width='1' fill='none' fill-rule='evenodd' fill-opacity='0'%3E%3Cg transform='translate(-249.000000, -126.000000)' fill='%23FFFFFF'%3E%3Crect x='249' y='126' width='1' height='1'%3E%3C/rect%3E%3C/g%3E%3C/g%3E%3C/svg%3E)

  

也正是因为这个原因，我又更认真看了这次的 KAT-Coder-Pro V2。

  

从「能写」到「能把事情做完」，如果只从代码生成能力来看，其实现在很多模型都已经过了及格线。

  

写函数、补代码、生成页面，这些能力已经变成基础能力，但一旦进入真实使用场景，问题就会暴露出来。

  

因为真实任务不是一步完成的，而是一个连续的过程。

  

比如你让模型做一个完整的小应用，它至少要经历这样一条链路：理解需求、拆解任务、生成页面、调用接口、管理状态、调整交互，甚至还要在过程中不断修正前面的错误。

![图片](data:image/svg+xml,%3C%3Fxml version='1.0' encoding='UTF-8'%3F%3E%3Csvg width='1px' height='1px' viewBox='0 0 1 1' version='1.1' xmlns='http://www.w3.org/2000/svg' xmlns:xlink='http://www.w3.org/1999/xlink'%3E%3Ctitle%3E%3C/title%3E%3Cg stroke='none' stroke-width='1' fill='none' fill-rule='evenodd' fill-opacity='0'%3E%3Cg transform='translate(-249.000000, -126.000000)' fill='%23FFFFFF'%3E%3Crect x='249' y='126' width='1' height='1'%3E%3C/rect%3E%3C/g%3E%3C/g%3E%3C/svg%3E)

  

KAT-Coder-Pro V2 这次做的一件比较关键的事情，是把「脚手架泛化能力」当成一个核心能力来设计。

  

简单来说，就是它不是只针对某一个工具或者某一个框架去优化，而是尽量在不同环境下都能稳定工作，不管你用的是 Claude Code、Cline、Kilo，还是 OpenCode，它都尽量保持一致的表现。

  

尤其是在 OpenClaw 这种多 Agent 框架下，这件事的难度其实是很高的。

  

我把它接入龙虾之后的真实表现，这一点在接入 OpenClaw 之后会更明显，就像我最近非常喜欢适用 AutoClaw 但是苦恼于没有性价比（效果好响应又快又便宜）更好的 Coding Plan。

![图片](data:image/svg+xml,%3C%3Fxml version='1.0' encoding='UTF-8'%3F%3E%3Csvg width='1px' height='1px' viewBox='0 0 1 1' version='1.1' xmlns='http://www.w3.org/2000/svg' xmlns:xlink='http://www.w3.org/1999/xlink'%3E%3Ctitle%3E%3C/title%3E%3Cg stroke='none' stroke-width='1' fill='none' fill-rule='evenodd' fill-opacity='0'%3E%3Cg transform='translate(-249.000000, -126.000000)' fill='%23FFFFFF'%3E%3Crect x='249' y='126' width='1' height='1'%3E%3C/rect%3E%3C/g%3E%3C/g%3E%3C/svg%3E)

  

KAT 的模型接入简直就是完美适配。

  

这次是从数据构建到训练流程，都围绕这种「长链路、多工具」的真实场景去做优化，甚至针对 OpenClaw 做了专项强化。

![图片](data:image/svg+xml,%3C%3Fxml version='1.0' encoding='UTF-8'%3F%3E%3Csvg width='1px' height='1px' viewBox='0 0 1 1' version='1.1' xmlns='http://www.w3.org/2000/svg' xmlns:xlink='http://www.w3.org/1999/xlink'%3E%3Ctitle%3E%3C/title%3E%3Cg stroke='none' stroke-width='1' fill='none' fill-rule='evenodd' fill-opacity='0'%3E%3Cg transform='translate(-249.000000, -126.000000)' fill='%23FFFFFF'%3E%3Crect x='249' y='126' width='1' height='1'%3E%3C/rect%3E%3C/g%3E%3C/g%3E%3C/svg%3E)

  

所以对我来说最直接的感受是，在 OpenClaw 类的场景里面，KAT-Coder-Pro V2 的多步任务的完成率更高，长链路执行更稳定，在高频调用和高并发场景下，也更少出现中断和异常。

![图片](data:image/svg+xml,%3C%3Fxml version='1.0' encoding='UTF-8'%3F%3E%3Csvg width='1px' height='1px' viewBox='0 0 1 1' version='1.1' xmlns='http://www.w3.org/2000/svg' xmlns:xlink='http://www.w3.org/1999/xlink'%3E%3Ctitle%3E%3C/title%3E%3Cg stroke='none' stroke-width='1' fill='none' fill-rule='evenodd' fill-opacity='0'%3E%3Cg transform='translate(-249.000000, -126.000000)' fill='%23FFFFFF'%3E%3Crect x='249' y='126' width='1' height='1'%3E%3C/rect%3E%3C/g%3E%3C/g%3E%3C/svg%3E)

  

换一种更直观的说法是，它更像一个「能把事情做完的人」，KAT-Coder-Pro V2 在这一块的表现，我自己的体感是跑更顺了。

  

当模型开始有审美判断，但如果只是工程能力的提升，其实还不足以让我觉得它有明显差异。

  

再说价格，它最近上线了一个 Coding Plan，认真看了一下之后，发现性价比是比较高的。

![图片](data:image/svg+xml,%3C%3Fxml version='1.0' encoding='UTF-8'%3F%3E%3Csvg width='1px' height='1px' viewBox='0 0 1 1' version='1.1' xmlns='http://www.w3.org/2000/svg' xmlns:xlink='http://www.w3.org/1999/xlink'%3E%3Ctitle%3E%3C/title%3E%3Cg stroke='none' stroke-width='1' fill='none' fill-rule='evenodd' fill-opacity='0'%3E%3Cg transform='translate(-249.000000, -126.000000)' fill='%23FFFFFF'%3E%3Crect x='249' y='126' width='1' height='1'%3E%3C/rect%3E%3C/g%3E%3C/g%3E%3C/svg%3E)

  

我们可以直接按使用频率选择套餐，不需要自己去折腾 API，也不需要做复杂接入，还不用担心费用超标的问题，价格定好的包月套餐，跟龙虾搭配就是很香。

![图片](data:image/svg+xml,%3C%3Fxml version='1.0' encoding='UTF-8'%3F%3E%3Csvg width='1px' height='1px' viewBox='0 0 1 1' version='1.1' xmlns='http://www.w3.org/2000/svg' xmlns:xlink='http://www.w3.org/1999/xlink'%3E%3Ctitle%3E%3C/title%3E%3Cg stroke='none' stroke-width='1' fill='none' fill-rule='evenodd' fill-opacity='0'%3E%3Cg transform='translate(-249.000000, -126.000000)' fill='%23FFFFFF'%3E%3Crect x='249' y='126' width='1' height='1'%3E%3C/rect%3E%3C/g%3E%3C/g%3E%3C/svg%3E)

  

所以我觉得对于很多想直接用起来的人来说，这个门槛会低很多。

  

如果你只是想体验或者做一些实际项目，这一块是完全可以直接上手的。

  

也正是因为这一点，我在一开始实测的时候，才会有那种感觉，速度和性价比，反而成了最后的加分项。

  

其实写到这里，模型能力本身已经讲得差不多了。

  

咱们不再只是用它写代码，而是在用它完成一件具体的事情。

  

比如做一个页面，做一份 PPT，甚至做一个完整的小产品。

  

KAT-Coder-Pro V2 至少让我看到一个趋势：模型不只是更强了，也开始变得更「成熟」。
