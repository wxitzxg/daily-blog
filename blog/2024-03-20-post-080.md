一、claude code 配置说明

我目前使用的claude code 接的是智谱的GLM-5 大模型，GLM-5 是文本模型，不支持图片识别，如果要想截图给claude code，让它理解图片内容的话，需要搭配智谱的视觉理解mcp进行使用。

1、添加视觉理解mcp

给claude code 添加智谱的视觉理解mcp，截图如下：

![图片](https://mmbiz.qpic.cn/sz_mmbiz_png/78CrPDzxXyF0He82WzneGkHAibWqEkLjTzyicmiaNtHLz8OYxQ72FhrZf4ECeQENCg5h6FI2x7IBKaMj2T88FOg0DiaTLqty9jZuYKgN5qVMxaE/640?wx_fmt=png&from=appmsg&watermark=1&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=0)

配置文本：

```perl
{  "args": [    "-y",    "@z_ai/mcp-server"  ],  "command": "npx",  "env": {    "Z_AI_API_KEY": "你的智谱api key",    "Z_AI_MODE": "VISION"  },  "type": "stdio"}
```

进入claude，查看mcp是否配置成功，

四、经验总结

1、自从使用了claude code后，我已经不再使用Tare工具了，自从上次系统架构图生成实验后，大模型又发展了8个月了，很多东西已经发生了翻天覆地的变化了，现在让ai生成一个好看的系统架构图已经不是什么难事了；

2、claude code 除了cli 命令行方式让普通人觉得不友好外，其他的都是优点；

3、如果想要画特定风格、配色、布局的功能图或系统架构设计图，你可以先上传一张参照图，先让大模型学习一下，再开始画图。

4、claude code 虽然是cli 的方式，但画图仍然也是扛扛的。
