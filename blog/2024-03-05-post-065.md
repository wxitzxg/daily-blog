## 二、 为什么选 Tushare？

Tushare 接口稳、数据格式规范，是目前最好的补充选择之一。

这样一来，看海量化系统就凑齐了 **MiniQMT + Tushare + Baostock** 的“三驾马车”：

- **MiniQMT（主力）** ：数据质量可以，实时性好，关键还免费，复权算法跟同花顺、通达信一致，平时回测和实盘还是首选它。
- **Tushare（新增）** ：不用开终端，有网就能下。周末券商维护或者没装 QMT 的时候，它就是不错的替补选择。虽然复权算法跟券商软件有点差别（跟 Baostock 一致），但作为回测数据源完全够用了，关键是他的分钟数据能下载到多年前的，也有基本面数据可以下载。缺点是需要付费。
- **Baostock（替补）** ：完全免费，不需要积分也不用注册。虽然数据不够全，但作为 Tushare 积分不够时的备用方案，还是很香的。

有了这三个数据源，基本上可以实现全天候覆盖了，不过对于同一个股票，集中数据源尽量不要混合补充，主要是他们的复权算法有所不同。

## 三、 技术实现：不只是简单的“下载保存”

接数据源这事儿，表面看就是“调接口 -> 存数据库”，但真做起来坑不少。为了让大家用得顺手，我在后台做了不少优化。

### 3.1 突破接口限制

用过 Tushare 的朋友都知道，它的接口单次请求有行数限制（通常是 8000 行）。如果直接拉 10 年的数据，肯定会缺斤少两。

最简单的办法是按年循环下，但那样太慢了。所以我写了一套 **智能分批算法** ：

- **日线数据** ：自动按约 4000 个交易日（大概 17 年）切一刀，既能一次下完，又不会触发限制。
- **分钟数据** ：根据你选的频率（比如 1 分钟线每天 241 根），自动计算每次能下多少天，卡着边界下，效率最大化。

这些复杂的计算都在后台自动完成，你只需要选好“2010年到2024年”，剩下的交给系统就行。

### 3.2 增量更新：拒绝重复劳动

数据下载最怕的就是“从头再来”。为了省时间，我加了 **智能增量检测** 。

下载前，系统会先看看你本地数据库里已经有了什么：

- 如果你有 2020-2022 年的数据，想更新到 2024 年，系统会自动识别，只下载 2023-2024 年的部分。
- 如果你有 1 月和 3 月的数据，唯独缺了 2 月，系统也会精准定位，只把 2 月的补上。

这种“缺哪补哪”的策略，能帮你省下大量的等待时间。

### 3.3 本地复权：省积分更省心

复权数据的处理是量化系统中的难点。传统的做法是直接向 API 请求“前复权”数据。但这有两个问题：

1. **消耗大** ：每次切换复权方式（如从前复权切到后复权）都要重新下载一遍。
2. **维护难** ：今天的前复权价格，到了明天因为发生了分红，历史价格就全变了，导致数据库里的旧数据全部失效。

在 V3.2 版本中，我采用了 **“原始数据 + 复权因子”** 的方案。

下载时，系统只下载 **不复权（Raw）** 的原始行情，并额外下载一份 **复权因子（Adj Factor）** 表。

当需要写入 DuckDB 时，系统会在本地内存中利用 Pandas 进行向量化计算，一次性生成不复权、前复权、后复权三种数据写入数据库。

这种方案的好处是巨大的： **你只需要下载一次数据，就可以拥有全复权的支持。** 既节省了 API 积分（复权因子接口通常开销很小），又保证了数据的灵活性。

### 4.1 数据源设置

在使用 Tushare 之前，需要先进行简单的配置：

1. 点主界面顶部的【设置】。
2. 切到【数据源设置】。
3. **Tushare 用户** ：填入你的 Token（去 Tushare 官网注册并获取）。
4. **Baostock 用户** ：无需配置，直接可用。
5. **配置数据存储路径** ：在【DuckDB数据路径】栏，选择一个空间充足的文件夹用于存放数据。
6. **代理设置** ：如果有必要，还可以配置 HTTP 代理。

数据源设置截图

配置完成后，当你使用相应的数据管理模块时，系统会自动调用这些配置。

### 4.2 选择合适的数据模块

看海量化系统为不同的数据源提供了独立的管理模块，你可以根据当前需求灵活选择：

- **MiniQMT 数据管理** ：当你打开 MiniQMT 数据管理模块时，系统会通过 MiniQMT 接口进行高速下载。适合盘中实时更新和通过券商终端获取精准数据。
- **Baostock 数据管理** ：无需任何配置，打开即用。虽然它主要支持日线数据，但作为完全免费且无需账号的“兜底”方案，在 Tushare 积分不足或需要快速验证日线策略时非常实用。
- **Tushare 数据管理** ：当你打开 Tushare 数据管理模块时，系统会完全基于 Tushare 接口工作。适合周末、节假日或未安装券商终端时的离线数据补充。

### 4.3 数据管理功能

无论你使用哪个模块，操作逻辑都是一致的：

- **选股票** ：可以直接选“沪深300”等板块，或导入 CSV。
- **选时间** ：精确到分钟，支持自定义时间段。
- **点补充** ：点击【补充数据】，系统会自动扫描本地数据库的缺口并进行精准填补。

### 4.4 数据校验

下完数据，直接在右边的【数据查询】里就能看。

- **多周期** ：Tick、分钟、日线都能看。
- **看复权** ：点一下复权下拉框，不复权/前复权/后复权数据立马切换显示。
- **看图表** ：双击数据行，还能弹个 K 线图出来，数据对不对一目了然。
