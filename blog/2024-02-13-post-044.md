## 每日一 Skills 推荐｜frontend-design + ui-ux-pro-max：让 AI 写出来的页面不再像 AI 写的

用 AI 写前端代码有一个很容易辨认的通病。

Inter 字体，紫色渐变，白色背景，圆角卡片三列排布，每张卡片上面一个 emoji 图标。你见过一次就会一直见到，因为 AI 生成的前端界面有一种高度趋同的“AI 味”。

行业里管这个叫 AI slop。

![配图 1：AI slop vs 有设计感的界面](https://mmbiz.qpic.cn/sz_mmbiz_jpg/meukczUWeTmMrHpZKoFXkRK9wejFz0ZyMjAwZHMytBialR0Pu543IRgk8kcAPjVH1icicxcVYb9HLkQg2sDxLqynVnwic20XicHUJzYkaLiaU3HYY/640?wx_fmt=jpeg&from=appmsg&watermark=1&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=0)

配图 1：AI slop vs 有设计感的界面

这两个 skill 就是来治这个病的。 `frontend-design` 来自 Anthropic 官方， `ui-ux-pro-max` 来自社区。一个管审美方向，一个管落地规范。思路不同，但目标一样：让 AI 生成的界面看起来像人设计的。

## frontend-design：反 AI slop 的审美指南

`frontend-design` 是 Anthropic 官方 skills 仓库里的一个。Anthropic 甚至专门写了一篇博客文章解释为什么要做这个 skill。

它的核心立场很明确：AI 生成的前端界面之所以难看，不是因为 AI 不会写代码，而是因为它总是收敛到统计意义上最“安全”的选择。Inter 是最常被推荐的字体，紫色渐变是最常见的 AI 配色，三列卡片是最常见的布局。每一个单独看都没问题，但组合在一起就成了“一看就是 AI 做的”。

`frontend-design` 的做法是在 skill 里明确列出禁区：

- 禁止使用 Inter、Roboto、Arial、系统字体
- 禁止紫色渐变 + 白色背景的配色
- 禁止没有上下文特征的通用布局
- 禁止千篇一律的组件模式

然后告诉 AI，在动手写代码之前，先想清楚四件事：

  

```
12345678910111213141516> 帮我做一个个人作品集网站
 
  [frontend-design 激活]
 
  设计方向分析：
  - 用途：展示设计/开发作品，吸引潜在雇主或客户
  - 调性：editorial / 杂志感，内容主导
  - 差异化记忆点：非对称布局 + 动态滚动交互
  - 字体选择：Playfair Display（展示标题）+ DM Sans（正文）
 
  配色方案：
  - 主色：炭黑 #1a1a1a
  - 强调色：暖铜 #c77d4a
  - 背景：米白 #f7f3ed
 
  开始生成代码...
```

它给 AI 提供了一个“审美决策框架”：先选一个极端的调性方向（brutally minimal、maximalist chaos、retro-futuristic、organic/natural、luxury/refined 等等），然后围绕这个方向做所有选择。字体要有个性，配色要有主次（60-70% 主色 + 1-2 个辅助色 + 一个强调色），布局要有意外感（非对称、重叠、对角线流动、打破网格），动效要集中火力（与其到处撒微交互，不如把一个页面加载动画做到位）。

它在 SKILL.md 里有一句话我觉得写得很到位：

`NEVER converge on common choices across generations.`

不要跨代收敛到同一种选择。每次生成的界面都应该不一样。

## ui-ux-pro-max：一本可以搜索的设计百科

如果说 `frontend-design` 是一个审美顾问， `ui-ux-pro-max` 更像一本工具书。

它内置了一个可搜索的设计知识库：50 多种 UI 风格、97 套配色方案、57 组字体搭配、99 条 UX 准则、25 种图表类型，支持 9 个技术栈。

你可以按需查询：

  

```
123456789101112131415161718192021222324252627> 帮我做一个金融科技的 SaaS 后台
 
  [ui-ux-pro-max 激活]
 
  python3 search.py "fintech SaaS dashboard" --design-system -p "FinFlow"
 
  ╔══════════════════════════════════╗
  ║     DESIGN SYSTEM: FinFlow      ║
  ╠══════════════════════════════════╣
  ║ PATTERN: SaaS Dashboard         ║
  ║ STYLE:  Minimalism + Dark Mode  ║
  ║                                 ║
  ║ COLORS:                         ║
  ║   Primary:  #0EA5E9 (sky-500)   ║
  ║   Surface:  #0F172A (slate-900) ║
  ║   Accent:   #22D3EE (cyan-400)  ║
  ║                                 ║
  ║ TYPOGRAPHY:                     ║
  ║   Heading:  Geist               ║
  ║   Body:     Geist               ║
  ║   Code:     JetBrains Mono      ║
  ║                                 ║
  ║ ANTI-PATTERNS:                  ║
  ║   ✗ Gradient backgrounds        ║
  ║   ✗ Rounded card excess         ║
  ║   ✗ Emoji as icons              ║
  ╚══════════════════════════════════╝
```

它的知识库按优先级排列：

| 优先级 | 类别 | 影响等级 |
| --- | --- | --- |
| 1 | 无障碍 | 最高 |
| 2 | 触控交互 | 最高 |
| 3 | 性能 | 高 |
| 4 | 响应式布局 | 高 |
| 5 | 字体配色 | 中 |
| 6 | 动效 | 中 |
| 7 | 风格选择 | 中 |
| 8 | 图表数据 | 低 |

无障碍排第一。这个排序本身就说明了它的价值取向：好看是重要的，但可用性更重要。

它还有一个“交付前检查清单”，在生成代码之前自动验证：

- 没有用 emoji 当图标（应该用 SVG）
- 所有可点击元素都有 cursor-pointer
- hover 状态不会导致布局抖动
- 亮色/暗色模式下对比度都够
- 移动端没有横向滚动
- 图片有 alt 文字
- 动效尊重 prefers-reduced-motion

这些规则一条一条看都不稀奇，但攒在一起就很有用。因为 AI 生成前端代码时最容易犯的错误，往往不是写不出来，而是漏掉这些细节。

## 两个放在一起看

这两个 skill 解决的问题有重叠，但切入点不同。

`frontend-design` 的重点是审美。它在说：别用那些无聊的默认选择了，做点有个性的东西出来。它更适合从零开始做一个新界面，需要有视觉冲击力的场景。作品集、着陆页、营销站、产品发布页这类。

`ui-ux-pro-max` 的重点是规范。它在说：这是 97 套配色方案、57 组字体搭配、99 条最佳实践，你按需取用。它更适合有明确业务需求的场景。SaaS 后台、电商页面、数据看板、管理系统这类。

两个可以同时装。 `frontend-design` 负责把审美标准拉高， `ui-ux-pro-max` 负责确保落地不踩坑。

安装：

  

```
12345# frontend-design（Anthropic 官方）
npx skills add anthropics/skills@frontend-design
 
# ui-ux-pro-max（社区）
npx skills add enuno/ui-ux-pro-max
```

## 它们不太适合什么

如果你做的是后端 API、命令行工具、数据管道这类不涉及界面的工作，这两个 skill 对你没用。

如果你的项目有严格的设计规范（公司设计系统、品牌手册）， `frontend-design` 的“大胆突破”风格可能和你的约束冲突。这种情况下 `ui-ux-pro-max` 更合适，它的 design system 持久化功能可以把你的规范写进去，后续生成的代码会遵循。

另外， `ui-ux-pro-max` 依赖 Python 脚本做知识库搜索。如果你的环境没有 Python，搜索功能跑不起来，不过 SKILL.md 本身的规则清单仍然有效。

## 为什么这篇值得放进“每日一 Skills 推荐”

AI slop 是一个真实存在的问题。

不只是前端。AI 生成的文本有 AI 味（我们的 humanizer-zh skill 就是治这个的），AI 生成的 PPT 有 AI 味（之前聊 PPTX skill 时专门提了 “Don’t create boring slides”）。前端界面的 AI slop 可能是最明显的，因为视觉上一眼就能认出来。

这个系列从文字润色到文档处理到代码生成，AI slop 的问题一路跟过来。 `frontend-design` 和 `ui-ux-pro-max` 是目前在前端领域对这个问题给出回应最认真的两个 skill。

前者从 Anthropic 内部自上而下推动，后者在社区里自下而上积累了 800 多个 star 和 57 组字体搭配。方向不同，要解决的问题是一样的。

## 最后

下次你让 AI 帮你写一个前端页面的时候，如果结果又是 Inter 字体 + 紫色渐变 + 三列卡片，你知道该装什么了。
