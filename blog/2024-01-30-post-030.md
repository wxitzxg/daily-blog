3 月 18 号，Google 给 Stitch 来了一次大更新。

加了个新概念叫 Vibe Design：不告诉 AI 画什么组件，告诉它你想让用户"感受到什么"。AI 自己决定配色、字体、布局。

免费，每月 350 次生成额度。消息出来当天 Figma 股价就跌了。

我之前用 Claude Code 分别做过 Web 端和移动端的 UI 设计，写了两篇文章。

现在 Google 这边出了新东西，自然要拉出来试试。

大家好，我是陆徐洲。

这次我用同一个中文提示词，测了四种方案。任务是给一款叫"墨流 InkFlow"的 AI 写作助手做中文落地页，内容包括导航栏、首屏、功能卡片、用户评价、数据展示、定价区、FAQ、底部导航。

先看看Stitch 的表现。

同一段提示词我连发了三次，三次风格完全不同。第一版绿色系白底，灵动大方。第二版预览有问题，只有文字排布没有 UI 效果，得手动点进编辑状态才正常。做过 Pencil 的朋友应该熟悉这种情况。第三版直接变暗黑风。

**Stitch 的设计审美确实在线。** 第一版的配色体系、版面节奏、留白处理都很舒服，不是那种"安全但无聊"的套路。两分钟出图，速度也碾压。

迭代测试我让它改三处布局。画布上看着像重新生成了一版，我一度以为内容丢了。导出 ZIP 打开 HTML 一看，三处修改全部到位。

**问题出在画布预览体验，设计和生成能力本身没毛病。**

不过 Stitch 有两个明显短板：同一个提示词每次结果不一样，以及导出只有 HTML/CSS，代码结构比较粗糙，拿去直接开发不太现实。

说回 Claude Code。

我先不装任何设计插件直接跑，Claude 自己调了 Pencil MCP，6 分 20 秒出结果。配色偏蓝黑，被"墨流"两个字带跑了，对比度低，设计感不强。

又装上 12.4 万安装量的官方 frontend-design skill 跑了一遍，4 分 37 秒。比裸跑好一些，整体更明亮，但提升有限。

**Claude Code 写代码靠谱，自己做设计决策就差点意思。**

转折来了。

Stitch 导出的 ZIP 里除了 HTML 和截图，还有一个 DESIGN.md。

我点开一看，不是什么"主色绿色、圆角 8px"的参数表。它是一整套设计体系。

起了个名字叫 "The Modern Calligrapher"，定位中国水墨画美学。禁用 1px 边框，用背景色层级差做分隔。字体双轨制：Noto Serif SC 管标题气质，Manrope 管正文效率。按钮渐变模拟"湿墨光泽"，动画要像"墨在水中晕开"。

Stitch 花两分钟想出来的这套东西，比 frontend-design skill 的设计指导具体十倍。

![图片](data:image/svg+xml,%3C%3Fxml version='1.0' encoding='UTF-8'%3F%3E%3Csvg width='1px' height='1px' viewBox='0 0 1 1' version='1.1' xmlns='http://www.w3.org/2000/svg' xmlns:xlink='http://www.w3.org/1999/xlink'%3E%3Ctitle%3E%3C/title%3E%3Cg stroke='none' stroke-width='1' fill='none' fill-rule='evenodd' fill-opacity='0'%3E%3Cg transform='translate(-249.000000, -126.000000)' fill='%23FFFFFF'%3E%3Crect x='249' y='126' width='1' height='1'%3E%3C/rect%3E%3C/g%3E%3C/g%3E%3C/svg%3E)

我直接把 DESIGN.md 丢给 Claude Code，让它严格按规范生成落地页。

4 分 38 秒。字体、排版、色彩层次明显提升，四组里视觉效果最好。

这次测试下来我最大的感受是，Stitch 和 Claude Code 各有一块很强的长板。

Stitch 强在设计直觉。给它一段描述，它能快速找到一个有品位的视觉方向，配色、字体、组件风格都想得很到位。这是目前 Claude Code 不管装什么 skill 都做不到的。

Claude Code 强在工程实现。给它一份清晰的设计规范，生成的代码结构好、可读性高、浏览器打开就能跑。这是 Stitch 导出的 HTML 做不到的。

**DESIGN.md 是连接两者的桥。** Stitch 把设计决策写成结构化的规范文件，Claude Code 拿着这份规范去写代码。设计归设计，代码归代码，各干各的强项。

操作很简单。打开 stitch.withgoogle.com，输入你的页面需求，选一版满意的导出 ZIP。把 ZIP 里的 DESIGN.md 丢给 Claude Code，让它按规范生成页面。

Stitch 当设计总监，Claude Code 当前端工程师。这套组合目前是我试过的、出活质量最高的工作流。
