聊聊这款模型本身。

KAT-Coder-Pro V2 是快手 KwaiKAT 团队做的，定位是旗舰级 Agentic Coding 模型。两个核心卖点：一个是审美能力突破，一个是为龙虾深度定制。

我翻了一下他们的技术报告，发现训练方法挺有意思的。

https://arxiv.org/abs/2603.27703

******#01******

**模型训练的新思路**

他们用了一个叫 Specialize-then-Unify 的范式。听起来很学术，其实思路很朴素：先培养专才，再合体成全才。

具体怎么做的呢。他们把 Agentic Coding 这件事拆成了 5 个独立的专家领域：

SWE 负责软件工程修复和开发，WebCoding 负责前端生成和审美，Terminal 负责命令行推理，WebSearch 负责联网搜索和信息整合，General 负责通用代码能力。

每个领域都单独构建数据、单独做监督微调、单独做强化学习，训练出 5 个专家模型。最后再通过一种叫 On-Policy Distillation 的方法，把 5 个专家的能力融合进一个模型里。

这个思路解决了一个很现实的问题：不同领域的训练信号是冲突的。SWE 任务需要长链条的代码编辑和测试验证，WebCoding 需要审美判断，Terminal 需要环境状态追踪。

一个模型同时学这些东西，很容易顾此失彼。先分开练，各自练到顶尖，再合体，效果就好很多。

再说说审美这块。

技术报告里用了一个词叫 Aesthetic Collapse，美学崩塌。说的是通用模型在面对低信息密度输入时，会退化成保守默认值。

低信息密度输入就是普通用户说的那种话：帮我做一个酷一点的页面，要有街头风。

模型听到这种话，不知道怎么办，就会退回到最安全的选择：蓝白配色，单列网格，规规矩矩。能用，但没有灵魂。

快手的解法很有意思。他们给同一个页面设计三种不同详细程度的描述：设计师版本一千多字，专业用户版本两三百字，普通用户版本只有五十字。三种描述对应同一个输出。

这样训练下来，模型就学会了从五十个字里推断出一千字的设计决策。用户说不清楚，模型帮着补全。

同时，对评测方式也进行了重新设计。

以前的标准考的是还原度，给一张参考图看 AI 抄得像不像。快手搞了一套让专业设计师盲测打分的体系，把审美拆成配色、字体、布局、动效等十个独立维度，每个维度单独评分。

评分标准也变了：出彩无瑕疵才是满分，鼓励原创，惩罚平庸。

结果是 Landing Page 场景 59.8 分，Slides 场景 57.6 分，数据可视化场景 67.6 分，三个场景全部第一。

******#02******

**龙虾适配**

说完审美，再说龙虾适配。

玩龙虾的人都知道，这东西的核心就是调用各种工具。给模型一个任务，它要自己判断该调哪些工具、按什么顺序调、工具之间怎么配合。

任务简单的时候还好。任务一复杂，链路一拉长，模型就容易犯迷糊。该调的没调，不该调的调了，顺序搞反，最后任务完不成。

另外就是框架差异。Claude Code 是一套玩法，OpenClaw 是另一套，OpenCode 又不一样。同一个模型，在这个框架上顺，换一个可能就卡。

KAT-Coder 在这块下了功夫。

他们搭了一套叫 KwaiEnv 的基础设施，能同时跑几万个沙箱。每个沙箱里都是真实的代码仓库、真实的依赖、真实的测试。模型就在这种环境里反复练，练的是复杂场景下的判断力和工具组合能力。

训练数据从十万多个 GitHub 仓库里来，覆盖 11 种编程语言，最后构建出超过 10 万个高难度样本。

还有一招我觉得挺聪明。训练的时候，同一个任务用不同的框架各跑一遍。Claude Code 跑一遍，OpenClaw 跑一遍，OpenCode 再跑一遍。模型见过各种框架长什么样，切换起来就顺了。

结果是 PinchBench 拿到 88.7 分。Claude Code、Cline、OpenCode、Kilo Code 这些主流工具也都能用。大家看下下面的数据，目前 KAT-Coder-V2 整体的效果进第一梯队了。

**写在最后**

快手官方也推出了 Coding Plan，大家可以试一下。趁着现在还能买到，各家好用的 Token 真的是稀缺资源。

说实话，我去年体验过一次快手的 Code 模型，当时只能说中规中矩，后来就没再上心，又跑回去玩 Claude 了。

这次是昨天在朋友圈看到字节一个哥们提到，快手的模型还挺能打的。我就切过去玩了一下，上面这些效果都是昨天新测的。

龙虾那边没办法直接给大家做案例对比，但我自己切过去之后，发现整体在任务链路上确实做得不错。龙虾本身就是 Agent，要执行长任务、要调用工具，这个方向模型确实需要专门优化。

但我觉得最重要的还是审美。

Anthropic 一直在提，品味对于做 AI 来说是最重要的。我现在越来越有感觉了。

大家想想，现在各种各样的 Skill 都在解决审美问题，但这些 Skill 其实是在做边边角角的事情，是额外的优化。

如果把模型当成一个人，这个人本身审美就很好，脚手架对它来说就是锦上添花。

但如果模型本身审美不行，再加提示词去限定、去拉，最后出来的效果也是硬拉上去的，不会真的好。

原生审美，这件事还是超级重要的。

推荐大家去试试。
