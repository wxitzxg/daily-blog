原创 月月鸟 *2026年3月2日 14:40*

在 skills.sh 上闲逛的时候，我刷到了一个有意思的技能：chinese-novelist-skill，专门用来写中文小说的（地址：https://skills.sh/penglonghuang/chinese-novelist-skill/chinese-novelist）。

简单说一下 skills.sh，它是一个 AI Agent 的技能市场，收录了各种给 Claude Code、Cursor、Copilot 这类工具用的 Skill，一条命令就能装上。

为什么我会对一个写小说的 Skill 感兴趣？

因为我之前就用 Claude Code 裸写过小说，体验说实话挺糟糕的。最大的问题是 **人物状态丢失** ，写到第五章主角的性格就飘了，配角的名字都能搞混。

上下文一长，模型就开始"失忆"，像一个记性不好的作者，每写一章都在重新认识自己的角色。

所以看到这个 Skill 号称能解决长篇创作中的连贯性问题，我立马就想试试。

## 5 个问题，15 个章节，一个小时，一部完整的小说

安装 Skill 很简单，把目录丢到 Claude Code 的 skills 文件夹就行。

启动之后，Skill 会逐一问你 5 个问题：题材、主角设定、性格、核心冲突、章节数。我用上下箭头选了选，定了一个 15 章的女性成长故事的题材。

然后，Claude Code 就开始写了。

**没有中断，没有卡顿，一章接一章地往外吐。**

因为是逐章生成的，我就坐在旁边"追更"。它写一章，我看一章。大概一个小时左右，15 章全部跑完，加起来将近 5 万字。

你可以想象一下那个画面：我泡了杯咖啡，坐在屏幕前，像追网文一样看 AI 现场给我写小说。只不过这个"作者"的更新速度是每几分钟一章。

这个体验挺奇妙的。以前用 Claude Code 裸写长篇，最多撑三四章就得人工介入，手动整理上下文，提醒模型"你别忘了这个角色是干什么的"。

而这个 Skill 内置了 **章节摘要自动追踪机制** ，每写完一章，自动把关键信息压缩成摘要传给下一章，所以人物状态基本没丢。

这就是 Skill 的价值。 **它帮模型记住了该记住的东西。**

就像给一个才华横溢但记性差的作者配了一个尽职的编辑，随时在旁边提醒：上一章谁说了什么，这个角色的口头禅是什么，那条伏笔还没收。

## 读完之后，我觉得它合格

看完之后我有点意外。

这部小说的质量比我预期的好。首先是 **没有很重的 AI 味道** ，人物有前后一致的行为逻辑，主线和支线也都能收住。

当然，如果跟市面上头部网文比，差距还是很明显的。节奏偏平，没有特别让人"停不下来"的段落，也缺少那种让读者上头的"爽点"设计。

但作为一个"AI 用一个小时写出来的小说"，我给它打 60 到 65 分。合格线以上。

然后一个念头就冒出来了： **要不投稿试试？**

反正小说都写完了，投出去看看真实市场的反馈，不比自己在那儿瞎评价强？

## 投稿七猫：从选平台到写申请书，全程 AI 代劳

既然想投稿，第一步是搞清楚投哪里。

我直接问了豆包：国内有哪些网文投稿平台，各自的政策是什么？

豆包列了一堆：起点、番茄、七猫……我对比了一下，最后选了 **七猫** 。

原因很简单，七猫有一个"保底"政策，就是说只要稿子被收了，就按字数给钱，不用等读者付费。对于一个纯粹想测试流程的人来说，这是最低风险的选择。

接下来要搞清楚七猫的投稿流程。

我打开七猫的投稿说明页面，用 Chrome 浏览器里内置的 Gemini 直接分析了这个页面，帮我梳理出了完整的投稿流程和注意事项。

投稿需要写一份申请书，包括作品简介、大纲、前几章样章这些。

这个我也没手写，直接用 Claude Code 配合 docx Skill，把之前生成的小说内容喂进去，让它提取申请书的填写字段并生成 Word 文档，顺便把投稿邮件也写好了。

**整个投稿准备过程，我没有手动写过一个字。**

检查了一遍，没啥问题，发送。

## 两天后，拒稿了

等了两天，收到了七猫的回复邮件。

打开一看，投稿未通过。

![图片](data:image/svg+xml,%3C%3Fxml version='1.0' encoding='UTF-8'%3F%3E%3Csvg width='1px' height='1px' viewBox='0 0 1 1' version='1.1' xmlns='http://www.w3.org/2000/svg' xmlns:xlink='http://www.w3.org/1999/xlink'%3E%3Ctitle%3E%3C/title%3E%3Cg stroke='none' stroke-width='1' fill='none' fill-rule='evenodd' fill-opacity='0'%3E%3Cg transform='translate(-249.000000, -126.000000)' fill='%23FFFFFF'%3E%3Crect x='249' y='126' width='1' height='1'%3E%3C/rect%3E%3C/g%3E%3C/g%3E%3C/svg%3E)

被拒了我倒不意外。仔细看了下编辑的反馈，原话是这么说的： **"阅读是流畅的，但情节比较常规，看点不够突出，叙述节奏有待提升，开篇吸引力不足。"**

拆开来看：

**情节比较常规，看点不够突出。** 这个我认。AI 生成的故事在"创意"这个维度上确实是弱项，它能写出结构完整的故事，但很难写出让编辑眼前一亮的设定。

**叙述节奏有待提升。** 网文有网文的规则，读者要的是节奏快、爽点密、代入感强。这些恰恰是 AI 目前不太擅长的，因为好的节奏背后是对读者心理的精准拿捏，光文字通顺远远不够。

**开篇吸引力不足。** 网文的"黄金三章"是铁律，前三章抓不住人，编辑就不会再往下看。AI 写的开头往往太"规矩"了，缺少那种一上来就让人挪不开眼的钩子。

值得注意的是，编辑说了"阅读是流畅的"。这说明 AI 在基本功层面已经过关了，文字本身没有硬伤。问题出在更高的层面：创意、节奏、吸引力。

但这些问题的根源， **是我压根没花时间去研究市场。**

我选题材的时候随手一选，确定大纲的时候也没有去分析七猫平台上什么类型的小说最受欢迎。

整个创作过程从头到尾不超过两个小时，包括安装 Skill、生成小说、准备投稿材料。

说白了，这不是一次认真的创作尝试，这是一次工具测试。

这次用到的所有素材，包括完整的小说、用到的 Skill 配置、投稿申请书，我都传到了 GitHub 上（地址：https://github.com/YuppyChen/novelist-test ），感兴趣的可以自己看看。

## 失败了，但整件事本身就是答案

回过头看这件事，我觉得最有意思的不是结果，是过程。

你想想整个链条：刷到 Skill，安装调试，生成完整小说，调研投稿平台，分析投稿流程，写申请书和邮件，最后提交投稿。

**所有"动手"的工作都是 AI 完成的。** 我做的事情就三件：提问题，选选项，点发送。

用到的工具拉个清单：

- • **Claude Code** ：核心工作台，所有操作都在终端里完成
- • **chinese-novelist-skill** ：小说创作的 Skill，解决了长篇写作中的上下文追踪问题
- • **豆包** ：信息咨询，帮我快速了解投稿平台的政策
- • **Chrome Gemini** ：网页分析，帮我梳理投稿流程
- • **docx Skill** ：文档生成，帮我撰写投稿申请书

五个工具，各司其职，串起了一个完整的从创作到投稿的流程。

这让我想到一个做菜的类比。以前你得自己买菜、切菜、炒菜、摆盘。

现在相当于你告诉 AI "我想吃一盘糖醋排骨"，它帮你把前面所有步骤都干了，你负责尝一口，决定要不要端上桌。

虽然这次投稿失败了，但我有一个比较确定的判断：

**只要愿意多花时间去研究市场上的爆款小说，把"爽点设计"和"节奏模板"投喂给模型，写出一部 70 分水平的长篇网文去投稿，问题不大。**

5 万字太短？那就写 20 万字，Skill 支持 50 章的创作，理论上完全可以做到。

立意不新颖？那就先多花点时间研究市面上流行的网文，找到读者真正爱看的类型和套路。

不够爽？那就把爆款小说的"爽点节奏"整理成参考资料，喂进 Skill 的创作模板里。

你看，这些都不是技术问题，是投入时间的问题。

**时代在变。** 今天 AI 能在一个小时里写完一部完整的小说，能帮你分析投稿平台的政策，能帮你撰写申请书和邮件。

不断探索模型和工具的能力上限，这本身就是当下 AI 的趋势。

重要的不是这次投稿有没有成功。

**重要的是，你知道了这条路走得通。**

下次再走，就知道该在哪里多花功夫了。

接下来我打算认真研究一下市面上的爆款网文，重新打磨一版升级版的小说，再去投稿看看效果。如果你对这个过程感兴趣，关注我，后续结果出来了第一时间分享。
