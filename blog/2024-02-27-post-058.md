## 神器本质：Token省在哪

RTK的核心逻辑很简单——做命令输出的"智能筛子"，在不影响AI理解的前提下，剔除冗余信息。它不替代任何命令，只是在输出传递给LLM的中间环节做优化，整个过程完全无感，却能让token消耗大幅下降。

其底层依赖Rust的高性能特性，启动延迟低于10ms，内存占用不足5MB，单二进制文件零依赖，安装后几乎不会影响原有开发流程。

## 4大核心压缩策略

### 智能过滤冗余

自动剔除对AI无意义的视觉噪音，包括终端颜色控制码、进度条动画、空白行、注释信息等。比如 `git push` 的原始输出有十几行，过滤后仅保留"ok main"核心结果，200 token直接压缩至10 token。

![图片](data:image/svg+xml,%3C%3Fxml version='1.0' encoding='UTF-8'%3F%3E%3Csvg width='1px' height='1px' viewBox='0 0 1 1' version='1.1' xmlns='http://www.w3.org/2000/svg' xmlns:xlink='http://www.w3.org/1999/xlink'%3E%3Ctitle%3E%3C/title%3E%3Cg stroke='none' stroke-width='1' fill='none' fill-rule='evenodd' fill-opacity='0'%3E%3Cg transform='translate(-249.000000, -126.000000)' fill='%23FFFFFF'%3E%3Crect x='249' y='126' width='1' height='1'%3E%3C/rect%3E%3C/g%3E%3C/g%3E%3C/svg%3E)

### 同类信息聚合

对大量同类输出做归类总结，避免重复堆砌。例如 `ls` 命令列出100个文件时，不会逐个罗列文件名，而是按目录和类型聚合为"src/目录下45个.java文件，test/目录下20个测试文件"这类摘要，2000 token可压缩至400 token。

### 关键信息截断

针对长日志输出，智能保留核心部分，砍掉冗余内容。比如测试命令执行时，仅汇总通过用例数量，完整保留失败用例的错误堆栈和上下文， `cargo test` 的25000 token可压缩至2500 token，节省比例达90%。

### 重复模式合并

识别并合并重复的输出结构，比如编译日志中反复出现的"Compiling..."、容器日志的时间戳前缀等，将重复内容合并为单一示意，避免无意义消耗

## 3分钟快速安装

### 支持平台

兼容macOS、Linux、Windows三大系统，适配Claude Code、Cursor、Codex、OpenCode等9大AI编程工具。

### 安装方式（三选一）

1. Homebrew（macOS推荐）：
```bash
brew install rtk
```
1. 一键脚本（Linux/macOS通用）：
```bash
curl -fsSL https://raw.githubusercontent.com/rtk-ai/rtk/refs/heads/master/install.sh | sh
```
1. Cargo安装（Rust开发者）：
```bash
cargo install --git https://github.com/rtk-ai/rtk
```

### 工具初始化

安装完成后，根据使用的AI工具执行对应初始化命令（-g表示全局生效）：

```bash
# Claude Code / GitHub Copilot
rtk init -g

# Codex
rtk init -g --codex

# OpenCode
rtk init -g --opencode

# Cursor
rtk init -g --agent cursor
```

重启AI编程工具即可生效，后续执行命令会自动被RTK处理，无需额外操作。

### 卸载方式

若需卸载，仅需一行命令：

```bash
rtk init -g --uninstall
```

## 常用功能与技巧

### 查看节省统计

内置统计命令，随时掌握token节省情况：

```bash
rtk gain                # 查看总体节省数据
rtk gain --graph         # 生成30天趋势图
rtk gain --daily         # 按天/周/月查看明细
rtk gain --project       # 查看当前项目节省情况
```

社区用户反馈，长期使用可累计节省千万级token，按当前定价计算，每月能节省不少开支。

### 自定义配置

执行 `rtk config --create` 可生成配置文件（路径：~/.config/rtk/config.toml），支持个性化设置：

- 命令排除：添加无需压缩的命令（如vim、nano等交互式工具）
- Tee模式：开启后自动保存失败命令的完整输出，方便调试
- 数据库路径：自定义统计数据存储位置

### 临时禁用技巧

调试复杂问题时需完整输出，可通过两种方式临时禁用RTK：

1. 直接使用原始命令，不加rtk前缀
2. 用命令完整路径调用： `/usr/bin/git status`
