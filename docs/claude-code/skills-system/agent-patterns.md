# Agent 模式完全指南

> 让 Claude Code 自主执行复杂任务

## 核心理念

Agent（代理）是 Claude Code 的核心能力——**AI 可以自己规划和执行任务**，而不只是响应单个指令。

## Agent vs 对话

| 对话模式 | Agent 模式 |
|----------|------------|
| 问一句答一句 | 自主规划执行 |
| 需要频繁指导 | 一次性任务描述 |
| 适合简单任务 | 适合复杂任务 |

**对话模式：**
```
用户: 帮我创建一个按钮
Claude: 好的，创建好了
用户: 改成蓝色
Claude: 改好了
用户: 再加个图标
Claude: 加好了
...
```

**Agent 模式：**
```
用户: 帮我重构用户认证模块，使用策略模式，支持多种登录方式

Agent:
1. 分析现有代码
2. 设计策略模式架构
3. 实现代码
4. 编写测试
5. 完成
```

## Agent 使用

### 基本语法

```
使用 Agent [任务描述]
```

### 示例

```
使用 Agent 模式重构用户认证模块：
1. 使用策略模式
2. 支持密码、短信、第三方登录
3. 保持向后兼容
4. 添加单元测试
```

## 最佳实践

### 1. 任务描述要清晰

**❌ 模糊**
```
帮我优化代码
```

**✅ 明确**
```
帮我重构 auth/service.ts：
- 使用策略模式
- 提取接口 IAuthStrategy
- 保持现有 API 兼容
- 添加单元测试覆盖
```

### 2. 设置检查点

对于复杂任务，让 Agent 定期停下来确认：

```
使用 Agent 开发用户模块：
- 第一步完成后确认
- 数据库设计确认后再实现
- 单元测试通过后再结束
```

### 3. 限定范围

**❌ 范围太大**
```
帮我重构整个后端
```

**✅ 范围可控**
```
帮我重构 user 服务（约 500 行代码）
```

## Subagent（子代理）

Claude Code 支持多代理协作：

```
主 Agent: 项目架构师
├── Subagent 1: 前端开发
├── Subagent 2: 后端开发
└── Subagent 3: 测试
```

**使用示例：**
```
使用 Agent 团队开发电商项目：
- 前端 subagent 负责 Vue 组件
- 后端 subagent 负责 API
- 测试 subagent 负责测试覆盖
```

## 常见问题

### Q: Agent 跑偏了怎么办？
A: 使用 `/stop` 停止，然后用 `/resume` 恢复或重新开始

### Q: 任务太复杂怎么办？
A: 拆分成多个小任务，用多个 Agent 调用

### Q: Agent 权限太大？
A: 在任务描述中明确限制范围
