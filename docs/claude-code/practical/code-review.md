# 代码审查流程

> 用 AI 进行专业的代码审查

## 核心理念

Claude Code 的 `/review` 命令是代码质量保障的利器。它可以：
- 快速审查代码
- 发现潜在问题
- 提供改进建议

## 使用方法

### 1. 基础审查

```
/review src/utils/helper.ts
```

### 2. 重点审查

```
/review src/utils/helper.ts
重点关注：
- 性能问题
- 内存泄漏
- 并发安全
```

### 3. 全面审查

```
/review src/
包括：
- 代码规范
- 安全漏洞
- 性能问题
- 测试覆盖
```

## 审查维度

| 维度 | 检查点 |
|------|--------|
| 代码规范 | 命名、格式、注释 |
| 性能 | 时间复杂度、空间复杂度 |
| 安全 | 注入、越权、泄露 |
| 可维护性 | 复杂度、耦合度 |
| 测试 | 覆盖率、边界情况 |

## 最佳实践

### 1. 审查前明确范围

```
审查 auth 模块，重点：
- 密码处理安全性
- JWT 实现正确性
- 权限控制逻辑
```

### 2. 结合团队规范

```
审查时参考 CLAUDE.md 中的规范：
- 组件不超过 300 行
- 函数不超过 50 行
- 必须有类型注解
```

### 3. 迭代改进

```
第一次审查：列出所有问题
第二次审查：确认修复
第三次审查：验证改进
```
