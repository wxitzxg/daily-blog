# 多模型协作

> 选择合适的模型完成特定任务

## 核心理念

Claude Code 支持多个模型，不同模型有不同的能力和价格。根据任务选择合适的模型，可以：
- 优化成本
- 提升效率
- 增强能力

## 模型选择

| 模型 | 能力 | 速度 | 价格 | 适用场景 |
|------|------|------|------|----------|
| Haiku | ⭐⭐ | 快 | 低 | 简单任务 |
| Sonnet | ⭐⭐⭐⭐ | 中 | 中 | 大部分任务 |
| Opus | ⭐⭐⭐⭐⭐ | 慢 | 高 | 复杂推理 |

## 使用场景

### 1. 日常对话 → Haiku

```
# 快速简单的问题
Haiku:
- 查文档
- 写简单函数
- 解释代码
```

### 2. 代码开发 → Sonnet

```
# 大部分开发工作
Sonnet:
- 写功能
- 重构代码
- 调试问题
```

### 3. 复杂推理 → Opus

```
# 需要深度思考
Opus:
- 系统设计
- 架构决策
- 复杂问题分析
```

## 切换模型

### 命令切换

```
切到 Opus 模型，帮我设计一个分布式系统
```

### 配置默认

```json
// ~/.claude/settings.json
{
  "model": {
    "default": "sonnet",
    "complex": "opus"
  }
}
```

## 实践技巧

### 1. 任务分配

```
用 Sonnet 写代码
用 Opus 审查架构
用 Haiku 查文档
```

### 2. 成本控制

```
简单任务用 Haiku
复杂任务用 Sonnet
关键决策用 Opus
```
