# 项目架构设计

> 用 AI 设计高质量的项目架构

## 核心理念

Claude Code 擅长分析和设计项目架构。利用它来：
- 评估现有架构
- 设计新系统
- 优化模块划分

## 使用方法

### 1. 让 AI 分析现有架构

```
分析 src 目录的架构，指出问题并给出优化建议
```

### 2. 设计新系统架构

```
设计一个电商系统架构，要求：
- 微服务架构
- 支持高并发
- 订单、支付、用户模块
```

### 3. 重构现有代码

```
重构 user 模块，从单体改为策略模式
```

## 架构模式参考

### 分层架构

```
┌─────────────────────────────────────┐
│           Presentation              │  ← Controller/API
├─────────────────────────────────────┤
│           Application               │  ← Service/UseCase
├─────────────────────────────────────┤
│             Domain                  │  ← Entity/Logic
├─────────────────────────────────────┤
│          Infrastructure             │  ← DB/External
└─────────────────────────────────────┘
```

### 整洁架构

```
/
├── domain/        # 核心业务逻辑
├── application/   # 用例
├── infrastructure/# 外部依赖
└── presentation/  # 接口
```

## 实践技巧

### 1. 提供上下文

```
分析电商订单模块的架构：
- 当前文件：src/modules/order/
- 技术栈：Node.js + TypeScript
- 痛点：订单处理逻辑混乱
```

### 2. 指定约束

```
设计时考虑：
- 团队 3 人
- 3 个月迭代周期
- 优先保证可维护性
```

### 3. 迭代优化

```
第一轮：给出初步架构
第二轮：根据反馈细化
第三轮：补充技术细节
```
